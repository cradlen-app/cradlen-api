Financial Module — Data Model

Three sub-modules, each owning a slice of the domain. Does this structure look right?

CORE/FINANCIAL/SERVICES
Service
id · UUID PK
organization_id · UUID? FK
code · String (unique/org)
name · String
description · String?
service_type · Enum
CONSULTATION | PROCEDURE
LAB_TEST | IMAGING | ADMIN | OTHER

is_deleted · Boolean

specialty_id · UUID FK
